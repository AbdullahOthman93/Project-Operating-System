# Project Operating System

**Status:** Architecture v1.0 approved (Phase 0 complete). Repository initialized (Sprint 1.1, Phase 1 in progress).

## What this is

A reusable, technology-agnostic operating framework that governs how AI models and software engineers plan, design, implement, validate, document, and maintain software projects. This repository is itself the first project governed by that framework (self-hosting).

## Where to start

- `docs/architecture/INDEX.md` — the Governance layer: principles, approval gates, decision rights. Start here.
- `docs/kernel/INDEX.md` — the Process layer: workflows and phase gates.
- `docs/contracts/INDEX.md` — the formal interfaces any tool or AI agent must satisfy to participate in this repository.

## Repository status

This repository currently contains structure and index documents only. Substantive content (principles, workflows, standards, contracts) is populated incrementally in later sprints, per the Incremental Development and Small Safe Changes principles in Architecture v1.0.

`src/` is intentionally absent and will be created only when implementation is required.

See `docs/architecture/INDEX.md` and the `INDEX.md` file in each `docs/` subdirectory for the purpose, scope, and dependencies of every part of this repository.
